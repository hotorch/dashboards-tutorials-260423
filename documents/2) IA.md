# IA 문서: 엠바고 요청 관리 도구

## 1. 사이트맵

```
/ (홈 = 대시보드)
├── 오늘 엠바고 요약
├── 엠바고 리스트
└── [모달] 신규 요청 생성
└── [모달] 요청 상세/수정
```

단일 페이지 구조입니다. 라우팅은 `/` 하나로 고정하고 모달로 세부 인터랙션을 처리합니다.

## 2. 유저 플로우

### 2.1 신규 요청 등록
```
대시보드 진입
→ 상단 [+ 신규 요청] 버튼 클릭
→ 모달 오픈 (제목, 설명, 엠바고 일시 입력)
→ 저장
→ 리스트 자동 갱신
```

### 2.2 상태 변경
```
리스트 행의 상태 드롭다운 클릭
→ 상태 선택 (요청됨 / 진행중 / 완료)
→ 즉시 저장 (낙관적 업데이트)
```

### 2.3 일일 현황 확인
```
대시보드 진입
→ 상단 요약 카드에서 오늘 건수 확인
→ 날짜 필터 "오늘" 클릭
→ 해당 리스트 확인
```

## 3. 네비게이션 구조

### 3.1 탑바 (Top Navigation)

| 영역 | 요소 | 동작 |
|---|---|---|
| 좌측 | 로고/타이틀 "엠바고 관리" | `/` 이동 |
| 중앙 | 없음 | - |
| 우측 | [+ 신규 요청] 버튼 | 생성 모달 오픈 |

단일 페이지이므로 탭이나 메뉴 항목은 없습니다.

### 3.2 콘텐츠 영역

```
┌─ Header (탑바) ─────────────────────┐
│                                     │
├─ 오늘 엠바고 요약 카드 ──────────────┤
│   [오늘 N건] [요청 N] [진행 N] [완료 N] │
│                                     │
├─ 필터 바 ───────────────────────────┤
│   [날짜: 오늘/주간/전체/커스텀]        │
│   [상태: 전체/요청/진행/완료]         │
│   [태그 필터]                        │
│                                     │
├─ 리스트 테이블 ─────────────────────┤
│   제목 | 엠바고 일시 | 상태 | PDF | 태그 | 액션 │
│   ...                               │
└─────────────────────────────────────┘
```

## 4. 페이지별 정보 구조

### 4.1 대시보드 (`/`)

#### 요약 카드 섹션
- 오늘 총 건수
- 상태별 카운트 (요청됨 / 진행중 / 완료)
- 다가오는 엠바고 (24시간 이내) 건수

#### 필터 바
- 날짜 프리셋: 오늘, 이번 주, 전체, 커스텀
- 상태 멀티 선택
- 태그 선택 (선택 기능)

#### 리스트 테이블
| 컬럼 | 설명 | 정렬 |
|---|---|---|
| 제목 | 요청 제목 | 가능 |
| 엠바고 일시 | `embargo_at` | 기본 오름차순 |
| 상태 | 드롭다운 (즉시 변경) | 가능 |
| PDF 발송 | 체크박스 | - |
| 태그 | 텍스트 | - |
| 액션 | 수정, 삭제 | - |

### 4.2 생성/수정 모달

| 필드 | 타입 | 필수 | 기본값 |
|---|---|---|---|
| 제목 | text input | O | - |
| 설명 | textarea | X | - |
| 엠바고 일시 | datetime-local | O | 현재 시각 |
| 상태 | select (수정 시) | O | `requested` |
| PDF 발송 | checkbox | X | false |
| 태그 | text input | X | - |

하단 버튼: `취소`, `저장`

## 5. 콘텐츠 분류 체계

### 5.1 상태 분류
```
requested  → 요청됨
in_progress → 진행중
completed  → 완료
```

색상 매핑 (신호등 톤):
- 요청됨: 빨강 (red) — 아직 처리 시작 전
- 진행중: 노랑 (amber) — 처리 중, 주의
- 완료: 초록 (emerald) — 처리 완료

### 5.2 날짜 필터 프리셋
- `today`: 오늘 00:00 ~ 23:59
- `this_week`: 이번 주 월~일
- `all`: 전체
- `custom`: 시작일, 종료일 직접 선택

### 5.3 태그 (선택 기능)
자유 입력 문자열. 별도 관리 페이지 없이 리스트에서 추가/선택합니다.

## 6. 인터랙션 패턴

| 상황 | 패턴 |
|---|---|
| 신규 생성 | 모달 + 저장 후 리스트 갱신 |
| 상태 변경 | 인라인 드롭다운, 낙관적 업데이트 |
| 삭제 | 확인 다이얼로그 후 실행 |
| PDF 체크 | 인라인 체크박스, 즉시 저장 |
| 필터 | URL 쿼리 파라미터 동기화 (선택) |

## 7. 반응형 전략 (데스크톱 우선)

| 해상도 | 처리 |
|---|---|
| ≥ 1024px | 기본 레이아웃, 테이블 전체 컬럼 노출 |
| 768 ~ 1023px | 요약 카드 2열, 테이블 스크롤 |
| < 768px | 테이블을 카드 리스트로 전환 (최소 대응) |

## 8. URL 구조

| URL | 용도 |
|---|---|
| `/` | 대시보드 (전체) |
| `/?date=today` | 오늘 필터 적용 |
| `/?status=in_progress` | 진행중 필터 |
| `/?date=today&status=requested` | 복합 필터 |

쿼리 파라미터로 필터 상태를 표현합니다. 별도 라우트는 두지 않습니다.

## 9. SEO 처리

내부 도구이므로 검색 엔진 노출은 불필요합니다.

- `robots.txt`: `Disallow: /`
- `<meta name="robots" content="noindex, nofollow">`
- 타이틀: "엠바고 관리" 고정

## 10. 접근성 최소 기준

- 모든 버튼에 `aria-label` 부여
- 모달 오픈 시 포커스 트랩
- 키보드 `Esc`로 모달 닫기
- 색상 외 텍스트/아이콘으로 상태 구분

## 11. 데이터 흐름 요약

```
[사용자 액션]
    ↓
[React 컴포넌트]
    ↓
[Supabase Client (supabase-js)]
    ↓
[Supabase REST API]
    ↓
[Postgres: embargo_requests]
    ↓
[응답 → 리스트/카드 실시간 갱신]
```

Realtime은 선택 사항입니다. MVP에서는 저장 후 수동 refetch로 충분합니다.
